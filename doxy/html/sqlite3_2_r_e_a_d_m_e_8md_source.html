11 ## Supported platforms
12 
13 The `sqlite3` module works with:
14 * Node.js v4.x, v6.x, v8.x, v10.x, v11.x, v12.x and v13.x.
15 * Electron v1.3.X, v1.6.x, v1.7.x, v1.8.x, v2.0.x, v3.0.x, v4.0.x, v4.1.x, v4.2.x, v5.0.x, v6.0.x, v6.1.x, v7.0.x, v7.1.x, v8.0.x, v8.1.x and v8.2.x
16 
17 Binaries for most Node versions and platforms are provided by default via [node-pre-gyp](https://github.com/mapbox/node-pre-gyp).
18 
19 The `sqlite3` module also works with [node-webkit](https://github.com/rogerwang/node-webkit) if node-webkit contains a supported version of Node.js engine. [(See below.)](#building-for-node-webkit)
20 
21 SQLite's [SQLCipher extension](https://github.com/sqlcipher/sqlcipher) is also supported. [(See below.)](#building-for-sqlcipher)

23 # Usage
24 
25 **Note:** the module must be [installed](#installing) before use.
26 
27 ``` js
28 var sqlite3 = require('sqlite3').verbose();
29 var db = new sqlite3.Database(':memory:');
30 
31 db.serialize(function() {
32  db.run("CREATE TABLE lorem (info TEXT)");
33 
34  var stmt = db.prepare("INSERT INTO lorem VALUES (?)");
35  for (var i = 0; i < 10; i++) {
36  stmt.run("Ipsum " + i);
37  }
38  stmt.finalize();
39 
40  db.each("SELECT rowid AS id, info FROM lorem", function(err, row) {
41  console.log(row.id + ": " + row.info);
42  });
43 });
44 
45 db.close();
46 ```

48 # Features
49 
50  - Straightforward query and parameter binding interface
51  - Full Buffer/Blob support
52  - Extensive [debugging support](https://github.com/mapbox/node-sqlite3/wiki/Debugging)
53  - [Query serialization](https://github.com/mapbox/node-sqlite3/wiki/Control-Flow) API
54  - [Extension support](https://github.com/mapbox/node-sqlite3/wiki/Extensions), including bundled support for the [json1 extension](https://www.sqlite.org/json1.html).
55  - Big test suite
56  - Written in modern C++ and tested for memory leaks
57  - Bundles Sqlite3 3.26.0 as a fallback if the installing system doesn't include SQLite

63 # Installing
64 
65 You can use [`npm`](https://github.com/isaacs/npm) to download and install:
66 
67 * The latest `sqlite3` package: `npm install sqlite3`
68 
69 * GitHub's `master` branch: `npm install https://github.com/mapbox/node-sqlite3/tarball/master`
70 
71 The module uses [node-pre-gyp](https://github.com/mapbox/node-pre-gyp) to download a pre-compiled binary for your platform, if it exists. Otherwise, it uses `node-gyp` to build the extension.
72 
73 It is also possible to make your own build of `sqlite3` from its source instead of its npm package ([see below](#building-from-the-source)).
74 
75 It is possible to use the installed package in [node-webkit](https://github.com/rogerwang/node-webkit) instead of the vanilla Node.js. See [Building for node-webkit](#building-for-node-webkit) for details.
76 
77 ## Source install
78 
79 To skip searching for pre-compiled binaries, and force a build from source, use
80 
81  npm install --build-from-source
82 
83 The sqlite3 module depends only on libsqlite3. However, by default, an internal/bundled copy of sqlite will be built and statically linked, so an externally installed sqlite3 is not required.
84 
85 If you wish to install against an external sqlite then you need to pass the `--sqlite` argument to `npm` wrapper:
86 
87  npm install --build-from-source --sqlite=/usr/local
88 
89 If building against an external sqlite3 make sure to have the development headers available. Mac OS X ships with these by default. If you don't have them installed, install the `-dev` package with your package manager, e.g. `apt-get install libsqlite3-dev` for Debian/Ubuntu. Make sure that you have at least `libsqlite3` >= 3.6.
90 
91 Note, if building against homebrew-installed sqlite on OS X you can do:
92 
93  npm install --build-from-source --sqlite=/usr/local/opt/sqlite/
94 
95 By default the node-gyp install will use `python` as part of the installation. A
96 different python executable can be specified on the command line.
97 
98  npm install --build-from-source --python=/usr/bin/python2
99 
100 This uses the npm_config_python config, so values in .npmrc will be honoured:
101 
102  python=/usr/bin/python2
103 
104 ## Custom file header (magic)
105 
106 The default sqlite file header is "SQLite format 3".
107 You can specify a different magic, though this will make standard tools and libraries unable to work with your files.
108 
109 
110  npm install --build-from-source --sqlite_magic="MyCustomMagic15"
111 
112 
113 Note that the magic *must* be exactly 15 characters long (16 bytes including null terminator).

115 ## Building for node-webkit
116 
117 Because of ABI differences, `sqlite3` must be built in a custom to be used with [node-webkit](https://github.com/rogerwang/node-webkit).
118 
119 To build node-sqlite3 for node-webkit:
120 
121 1. Install [`nw-gyp`](https://github.com/rogerwang/nw-gyp) globally: `npm install nw-gyp -g` *(unless already installed)*
122 
123 2. Build the module with the custom flags of `--runtime`, `--target_arch`, and `--target`:
124 
125 ```sh
126 NODE_WEBKIT_VERSION="0.8.6" # see latest version at https://github.com/rogerwang/node-webkit#downloads
127 npm install sqlite3 --build-from-source --runtime=node-webkit --target_arch=ia32 --target=$(NODE_WEBKIT_VERSION)
128 ```
129 
130 This command internally calls out to [`node-pre-gyp`](https://github.com/mapbox/node-pre-gyp) which itself calls out to [`nw-gyp`](https://github.com/rogerwang/nw-gyp) when the `--runtime=node-webkit` option is passed.
131 
132 You can also run this command from within a `node-sqlite3` checkout:
133 
134 ```sh
135 npm install --build-from-source --runtime=node-webkit --target_arch=ia32 --target=$(NODE_WEBKIT_VERSION)
136 ```
137 
138 Remember the following:
139 
140 * You must provide the right `--target_arch` flag. `ia32` is needed to target 32bit node-webkit builds, while `x64` will target 64bit node-webkit builds (if available for your platform).
141 
142 * After the `sqlite3` package is built for node-webkit it cannot run in the vanilla Node.js (and vice versa).
143  * For example, `npm test` of the node-webkit's package would fail.
144 
145 Visit the “[Using Node modules](https://github.com/rogerwang/node-webkit/wiki/Using-Node-modules)” article in the node-webkit's wiki for more details.

147 ## Building for sqlcipher
148 
149 For instructions for building sqlcipher see
150 [Building SQLCipher for node.js](https://coolaj86.com/articles/building-sqlcipher-for-node-js-on-raspberry-pi-2/)
151 
152 To run node-sqlite3 against sqlcipher you need to compile from source by passing build options like:
153 
154  npm install sqlite3 --build-from-source --sqlite_libname=sqlcipher --sqlite=/usr/
155 
156  node -e 'require("sqlite3")'
157 
158 If your sqlcipher is installed in a custom location (if you compiled and installed it yourself),
159 you'll also need to to set some environment variables:
160 
161 ### On OS X with Homebrew
162 
163 Set the location where `brew` installed it:
164 
165  export LDFLAGS="-L`brew --prefix`/opt/sqlcipher/lib"
166  export CPPFLAGS="-I`brew --prefix`/opt/sqlcipher/include"
167  npm install sqlite3 --build-from-source --sqlite_libname=sqlcipher --sqlite=`brew --prefix`
168 
169  node -e 'require("sqlite3")'
170 
171 ### On most Linuxes (including Raspberry Pi)
172 
173 Set the location where `make` installed it:
174 
175  export LDFLAGS="-L/usr/local/lib"
176  export CPPFLAGS="-I/usr/local/include -I/usr/local/include/sqlcipher"
177  export CXXFLAGS="$CPPFLAGS"
178  npm install sqlite3 --build-from-source --sqlite_libname=sqlcipher --sqlite=/usr/local --verbose
179 
180  node -e 'require("sqlite3")'
181 
182 ### Custom builds and Electron
183 
184 Running sqlite3 through [electron-rebuild](https://github.com/electron/electron-rebuild) does not preserve the sqlcipher extension, so some additional flags are needed to make this build Electron compatible. Your `npm install sqlite3 --build-from-source` command needs these additional flags (be sure to replace the target version with the current Electron version you are working with):
185 
186  --runtime=electron --target=1.7.6 --dist-url=https://electronjs.org/headers
187 
188 In the case of MacOS with Homebrew, the command should look like the following:
189 
190  npm install sqlite3 --build-from-source --sqlite_libname=sqlcipher --sqlite=`brew --prefix` --runtime=electron --target=1.7.6 --dist-url=https://electronjs.org/headers
