22 ## Install
23 
24 To install the stable version use the `Menu - Manage palette` option and search for `node-red-dashboard`, or run the following command in your Node-RED user directory - typically `~/.node-red`:
25 
26  npm i node-red-dashboard
27 
28 Restart your Node-RED instance and you should have UI nodes available in the palette and a new `dashboard` tab in the
29 right side panel. The UI interface is available at <http://localhost:1880/ui> (if the default settings are used).
30 
31 If you want to try the latest version from github, you can install it by
32 
33  npm i node-red/node-red-dashboard

35 ## Settings
36 
37 The default url for the dashboard is based off your existing Node-RED httpRoot path with /ui added. This can be changed in your Node-RED settings.js file - `ui: { path: "ui" },`
38 
39 You can also add your own express middleware to handle requests by using the `ui: { middleware: your_function }` property in settings.js. For example
40 
41 ```
42 ui: { path: 'ui', middleware: function (req, res, next) {
43  // Do something more interesting here.
44  console.log('LOGGED')
45  next()
46  }
47  },
48 ```
49 
50 You can also set the dashboard to be read only by `ui: { readOnly: true }`. This does not stop the user interacting with the dashboard but does ignore all updates coming from the dashboard.

52 ## Layout
53 
54 The dashboard layout should be considered as a grid.
55 
56 Each **group** element has a width - by default 6 'units' (a unit is 48px wide by default with a 6px gap).
57 
58 Each **widget** in the group also has a width - by default, 'auto' which means it will fill the width of the group it is in, but you can set it to a fixed number of units.
59 
60 The layout algorithm of the dashboard always tries to place items as high and to the left as they can within their container - this applies to how groups are positioned on the page, as well as how widgets are positioned in a group.
61 
62 Given a group with width 6, if you add six widgets, each with a width of 2, then they will be laid out in two rows - three widgets in each.
63 
64 If you add two groups of width 6, as long as your browser window is wide enough, they will sit alongside each other. If you shrink the browser, at some point the second group will shift to be below the first, in a column.
65 
66 It is advisable to use multiple groups if possible, rather than one big group, so that the page can dynamically resize on smaller screens.

68 ## Features
69 
70 #### Dashboard sidebar
71 
72 The widget layout is managed by a `dashboard` tab in the sidebar of the Node-RED editor.
73 
74 ##### Layout
75 
76  - **Tabs** - From here you can re-order the tabs, groups and widgets, and add and edit their properties. You can also open the layout tools that may help you organise the widgets more easily than via the sidebar.
77 
78  - **Links** - to other web pages can also be added to the menu. They can optionally be opened in an iframe - if allowed by the target page.
79 
80 ##### Site
81 
82  - **Title** - the `title` of the UI page can be set.
83 
84  - **Options** - optionally hide the title bar, and allow swiping sideways between tabs on a touch screen. You can also set whether the template uses the selected theme or uses the underlying Angular Material theme. You can also choose to use the Angular Material theme everywhere.
85 
86  - **Date Format** - sets the default date format for chart and other labels.
87 
88  - **Sizes** - sets the basic geometry of the grid layout in pixels. The **width** and **height** of widgets can be set, as can the width of *groups*. These are the basic definitions of the "units' used elsewhere within the dashboard.
89 
90 ##### Theme
91 
92  - **Style** - the theme and font of the UI is set in the dashboard sidebar. You can select a default Light, Dark or Custom Theme. You cannot have different themes for each tab.
93 
94 You can also choose to use the basic Angular Material themes instead if you like, either just within any ui_templates or for the whole Dashboard. This will only affect angular components so some of the charts and so on may need extra work.
95 
96 **Note**: For users creating their own templates the following CSS variable names are available
97 to help pick up the theme colours.
98 
99  - --nr-dashboard-pageBackgroundColor
100  - --nr-dashboard-pageTitlebarBackgroundColor
101  - --nr-dashboard-pageSidebarBackgroundColor
102  - --nr-dashboard-groupBackgroundColor
103  - --nr-dashboard-groupTextColor
104  - --nr-dashboard-groupBorderColor
105  - --nr-dashboard-widgetColor
106  - --nr-dashboard-widgetTextColor
107  - --nr-dashboard-widgetBgndColor

109 #### Widgets
110 
111 Don't forget there are also extra ui widgets available on the [Node-RED flows](http://flows.nodered.org) website. Search for node-ui- or contrib-ui- .
112 
113 Group labels are optional.
114 
115 Most widgets can have a label and value - both of these can be specified by properties of the incoming msg if required, and modified by angular filters. For example the label can be set to `{{msg.topic}}`, or the value could be set to `{{value | number:1}}%` to round the value to one decimal place and append a % sign.
116 
117 Each node may parse the `msg.payload` to make it suitable for display. This converted version is exposed as the variable called `value`, (see example above).
118 
119 Any widget can be disabled by passing in a `msg.enabled` property set to `false;`. *Note:* this doesn't stop the widget receiving messages but does stop inputs being active and does re-style the widget.
120 
121 Most ui widgets can also be configured by using a `msg.ui_control` message - see **[config-fields.md](https://github.com/node-red/node-red-dashboard/blob/master/config-fields.md)**
122 for futher details.
123 
124  - **Audio out** - a widget that will let you play audio (wav or mp3) or send Text to Speech (TTS) to the client.
125  - **Button** - the icon can be set using either Material or fa-icons - the colour and background colour may also be set. If the widget is sized to 1 wide the icon has precedence.
126  - **Chart** - has both line, bar and pie chart modes. Also the X-Axis labels can be customised using a date formatter string. See **[this document](https://github.com/node-red/node-red-dashboard/blob/master/Charts.md)** for more information on the chart data formats accepted.
127  - **Colour Picker** - a colour picker widget.
128  - **Date Picker** - a date picker widget. The displayed Date format can be specified in the Site tab using moment.js formatting.
129  - **Dropdown** - a dropdown select widget has been added. Multiple label, value pairs can be specified. The choices can also be set via `msg.options` containing an array of objects. If just text then the value will be the same as the label, otherwise you can specify both by using an object of "label":"value" pairs :
130 
131  [ "Choice 1", "Choice 2", {"Choice 3": 3} ]
132 
133  Setting `msg.payload` will pre-select the value in the dropdown.
134  - **Form** - a widget that can be composed of several sub-widgets. When submitted all values are submitted as a single message.
135  - **Gauge** - has 4 modes - *standard* (simple gauge), *donut* (complete 360&deg;), *compass*, and *wave*. You can also specify the colour range of the standard and donut gauges.
136  - **Notification** - creates alerts to the user - can either be a toast popup, or a dismissable alert box. The alert may be targeted to a single user.
137  - **Numeric** - a Numeric input widget with up/down buttons.
138  - **Slider** - a simple horizontal slider, with variable step size.
139  - **Switch** - can also set two icons and/or colours depending on state.
140  - **Template** - the template node allows the user to specify and create their own widgets within the framework using HTML, Javascript. This is an Angular.js widget. You may also use this to override the built in CSS styles.
141  - **Text** - A read only widget, the layout of the `label`, and `value` can be configured.
142  - **Text input** - text input box, with optional label, can also support password, email and colour modes.
143  - **UI-Control** - allows some dynamic control of the dashboard. Sending a `msg.payload` of the tab number (from 0) or tab_name will switch to that tab. Tabs can be enabled/disabled/hide/show via msg like `{"tabs":{"hide":["tab_name_with_underscores"],"show":["another_tab_name"],"disable":["unused_tab_name"]}}`.
144  Groups can be hidden and made visible via a msg like `{"group":{"hide":["tab_name_group_name_with_underscores"],"show":["tab_name_another_group"],"focus":true}}`. Outputs a `msg.payload` for every browser *connect* and *loss*, and every tab *change*. This can be used to trigger other actions like resetting the visibility of tabs and groups.
145 
146 **Tip:** The *Text* widget will accept html - so you can use it together with the *fa-icons* we
147 already use to create indicator type widgets.

149 ## Icons
150 
151 The dashboard has 4 sets of icons built in. They are
152 
153  - [Angular Material icons](https://klarsys.github.io/angular-material-icons/) : e.g. `send`
154  - [Font Awesome 4.7](https://fontawesome.com/v4.7.0/icons/) : e.g. `fa-fire fa-2x`
155  - [Weather Icons Lite](https://github.com/Paul-Reed/weather-icons-lite/blob/master/css_mappings.md) : e.g. `wi-wu-sunny`
156  - [Material Design Iconfont](https://jossef.github.io/material-design-icons-iconfont/) ; e.g. `mi-alarm_on` - note add mi- to the icon name in the iconset.
157 
158 And 1 that can only be used if you have a permanent connection to the internet
159 
160  - [Iconify](https://iconify.design/icon-sets/) : e.g. `iconify-mdi:car-battery 48px`
161 
162 Again note you have to add `iconify-` to the icon name in the icon set of your choice. You may also optionally specify a size in standard px or em notation. Default is 24px. You must add a **ui_template** node that loads the necessary iconify library into the header of the dashboard. It should contain
163 
164 ```
165 <script src="https://code.iconify.design/1/1.0.7/iconify.min.js"></script>
166 ```
167 
168 You may also create your own set of icons using [Icofont](https://icofont.com/icons). Once downloaded you can serve them locally via Node-RED and add them to the head of the dashboard page by using a ui_template node : e.g.
169 
170 ```
171 <link rel="stylesheet" href="myserver/path/icofont.css">
172 ```
173 then you can use then as per above by adding the icofont- prefix e.g. `icofont-badge`
174 
175 Or just use them in a template
176 
177 ```
178 <link rel="stylesheet" href="/css/icofont.css">
179 <div style="display: flex;height: 100%;justify-content: center;align-items: center;">
180 <i class="icofont icofont-4x icofont-hail"></i>
181 </div>
182 ```
183 
184  And finally via an image - https://{myserver/path/image.png}
185 
186 You can use them in any of the Icon fields. You may also be able to use some of them for example in labels via their inline style eg `<i class="fa fa-flag"></i>`

189 ## Loading the Dashboard
190 
191 Due to the size of the dashboard libraries it can take a long time to load if you are running on wireless network. It is possible add a custom loading page if you wish. To do so add a file called `loading.html` to the `node_modules/node-red-dashboard/dist/` folder. A simple example could be
192 
193 ```
194  <div><i class="fa fa-spin fa-5x fa-spinner"></i></div>
195  ```

197 ## Securing the Dashboard
198 
199 You can use the `httpNodeAuth` property in your Node-RED settings.js file to secure the Dashboard as it is
200 created the same way as other HTTP nodes are. The details can be found at the bottom of this page in the
201 docs <http://nodered.org/docs/security>

203 ## Multiple Users
204 
205 This Dashboard does NOT support multiple individual users. It is a view of the status of the underlying
206 Node-RED flow, which itself is single user. If the state of the flow changes then all clients will get
207 notified of that change.
208 
209 Messages coming from the dashboard **do** have a `msg.socketid`, and updates like change of tab,
210 notifications, and audio alerts will be directed only to that session. Delete the `msg.sessionid` to send
211 to all sessions.
