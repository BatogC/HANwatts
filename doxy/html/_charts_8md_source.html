1 # Node-RED-Dashboard Charts
2 
3 ## Live data
4 
5 To display live data just wire up an input with a `msg.payload` that contains a number.
6 
7 
8 ### Line charts
9 
10 To display two or more lines on the same chart then each `msg` must also contain `topic`
11 property that identifies which data series it belongs to - for example:
12 
13  {topic:"temperature", payload:22}
14  {topic:"humidity", payload:66}
15 
16 Each series will be represented by a different colour.
17 Alternatively you can use the property `series` instead of `topic` if you prefer.
18 
19 It is possible to create "gaps" in line charts by sending either a null or boolean false as the payload.
20 
21 You can also insert extra data points by specifying the `timestamp` property. This must be in
22 either epoch time (in miliseconds, not seconds), or ISO8601 format.
23 
24  {topic:"temperature", payload:22, timestamp:1520527095000}
25 
26 ### Bar, and other charts
27 
28 If you want all the bars to be the same colour, then use the `label` property instead.
29 
30  {label:"indoor temperature", payload:22}
31  {label:"outdoor temperature", payload:15}
32 
33 You can have both a label and series property if you want.
34 
35 ## Stored data
36 
37 To display a complete chart in one go - for example from a set of points retrieved from a database,
38 the data must be supplied in the form of an array, that holds an object that has `series`,`labels`, and `data` arrays.
39 This is broadly the same as the raw format used by the angular chart.js library.
40 
41 You will need to process your data into this structure in order to render it correctly.
42 
43 ### Line charts
44 
45 For line charts an example is given below
46 
47  [{
48  "series": ["A", "B", "C"],
49  "data": [
50  [{ "x": 1504029632890, "y": 5 },
51  { "x": 1504029636001, "y": 4 },
52  { "x": 1504029638656, "y": 2 }
53  ],
54  [{ "x": 1504029633514, "y": 6 },
55  { "x": 1504029636622, "y": 7 },
56  { "x": 1504029639539, "y": 6 }
57  ],
58  [{ "x": 1504029634400, "y": 7 },
59  { "x": 1504029637959, "y": 7 },
60  { "x": 1504029640317, "y": 7 }
61  ]
62  ],
63  "labels": [""]
64  }]
65 
66 For non-timeseries data - for example data in labelled categories, rather than use x and y,
67 the format should be as follows:
68 
69  [{
70  "series": ["X", "Y", "Z" ],
71  "data": [ [5,6,9,10], [3,8,5,10], [6,7,2,10] ],
72  "labels": [ "Jan", "Feb", "Mar", "Apr" ]
73  }]
74 
75 ### Bar and Pie charts
76 
77 for bars of different colours
78 
79  [{
80  "series": [ "X", "Y", "Z"],
81  "data": [ [5], [3], [6] ],
82  "labels": [ "Jan" ]
83  }]
84 
85 for bars of the same colour, set the flag `Use first colour for all bars` in the
86 node configuration, and set labels for each column
87 
88  [{
89  "series": [ "X" ],
90  "data": [ [5,6,9] ],
91  "labels": [ "Jan", "Feb", "Mar" ]
92  }]
93 
94 and these can be mixed - you can have series and labels defined...
95 
96  [{
97  "series": ["X", "Y", "Z" ],
98  "data": [ [5,6,9,10], [3,8,5,11], [6,7,2,12] ],
99  "labels": [ "Jan", "Feb", "Mar", "Apr" ]
100  }]
