63 ## Options
64 
65 - `acceptWidgets` - accept widgets dragged from other grids or from outside (default: `false`). Can be:
66  * `true` (uses `'.grid-stack-item'` class filter) or `false`
67  * string for explicit class name
68  * function (i: number, element: Element) returning a boolean. See [example](http://gridstack.github.io/gridstack.js/demo/two.html)
69 - `alwaysShowResizeHandle` - if `true` the resizing handles are shown even if the user is not hovering over the widget
70  (default: `false`)
71 - `animate` - turns animation on (default: `false`)
72 - `auto` - if `false` gridstack will not initialize existing items (default: `true`)
73 - `cellHeight` - one cell height (default: `60`). Can be:
74  * an integer (px)
75  * a string (ex: '100px', '10em', '10rem', '10%')
76  * 0 or null, in which case the library will not generate styles for rows. Everything must be defined in CSS files.
77  * `'auto'` - height will be calculated to match cell width (initial square grid).
78 - `column` - number of columns (default: `12`) which can change on the fly with `setColumn()` as well. See [example](http://gridstackjs.com/demo/column.html)
79 - `ddPlugin` - class that implement drag'n'drop functionallity for gridstack. If `false` grid will be static. (default: `null` - first available plugin will be used)
80 - `disableDrag` - disallows dragging of widgets (default: `false`).
81 - `disableOneColumnMode` - disables the onColumnMode when the window width is less than minWidth (default: 'false')
82 - `disableResize` - disallows resizing of widgets (default: `false`).
83 - `draggable` - allows to override jQuery UI draggable options. (default: `{handle: '.grid-stack-item-content', scroll: false, appendTo: 'body', containment: null}`)
84 - `dragOut` to let user drag nested grid items out of a parent or not (default false) See [example](http://gridstackjs.com/demo/nested.html)
85 - `float` - enable floating widgets (default: `false`) See [example](http://gridstackjs.com/demo/float.html)
86 - `handle` - draggable handle selector (default: `'.grid-stack-item-content'`)
87 - `handleClass` - draggable handle class (e.g. `'grid-stack-item-content'`). If set `handle` is ignored (default: `null`)
88 - `itemClass` - widget class (default: `'grid-stack-item'`)
89 - `maxRow` - maximum rows amount. Default is `0` which means no maximum rows
90 - `minWidth` - minimal width. If window width is less than or equal to, grid will be shown in one-column mode (default: `768`)
91 - `oneColumnModeDomSort` - set to `true` if you want oneColumnMode to use the DOM order and ignore x,y from normal multi column layouts during sorting. This enables you to have custom 1 column layout that differ from the rest. (default?: `false`)
92 - `placeholderClass` - class for placeholder (default: `'grid-stack-placeholder'`)
93 - `placeholderText` - placeholder default content (default: `''`)
94 - `resizable` - allows to override jQuery UI resizable options. (default: `{autoHide: true, handles: 'se'}`)
95 - `removable` - if `true` widgets could be removed by dragging outside of the grid. It could also be a jQuery selector string, in this case widgets will be removed by dropping them there (default: `false`) See [example](http://gridstackjs.com/demo/two.html)
96 - `removeTimeout` - time in milliseconds before widget is being removed while dragging outside of the grid. (default: `2000`)
97 - `rtl` - if `true` turns grid to RTL. Possible values are `true`, `false`, `'auto'` (default: `'auto'`) See [example](http://gridstackjs.com/demo/rtl.html)
98 - `staticGrid` - makes grid static (default `false`). If true widgets are not movable/resizable. You don't even need jQueryUI draggable/resizable. A CSS class `grid-stack-static` is also added to the container.
99 - `verticalMargin` - vertical gap size (default: `20`). Can be:
100  * an integer (px)
101  * a string (ex: '2em', '20px', '2rem')

103 ## Grid attributes
104 
105 - `data-gs-animate` - turns animation on
106 - `data-gs-column` - amount of columns. Setting non-default value must be supported by equivalent change in CSS, [see docs here](https://github.com/gridstack/gridstack.js#change-grid-columns).
107 - `data-gs-max-row` - maximum rows amount. Default is `0` which means no maximum rows.
108 - `data-gs-current-height` - current rows amount. Set by the library only. Can be used by the CSS rules.
109 
110 ## Item attributes
111 
112 - `data-gs-x`, `data-gs-y` - (number) element position in row/column. Note: if one is missing this will `autoPosition` the item
113 - `data-gs-width`, `data-gs-height` - (number) element size in row/column
114 - `data-gs-id`- (number | string) good for quick identification (for example in change event)
115 - `data-gs-max-width`, `data-gs-min-width`, `data-gs-max-height`, `data-gs-min-height` - element constraints in row/column
116 - `data-gs-no-resize` - disable element resizing
117 - `data-gs-no-move` - disable element moving
118 - `data-gs-auto-position` - tells to ignore `data-gs-x` and `data-gs-y` attributes and to place element to the first available position. Having either one missing will also do that.
119 - `data-gs-locked` - the widget will be locked. It means another widget wouldn't be able to move it during dragging or resizing.
120 The widget can still be dragged or resized. You need to add `data-gs-no-resize` and `data-gs-no-move` attributes
121 to completely lock the widget.
122 - `data-gs-resize-handles` - sets resize handles for a specific widget.

226 ### addWidget(el, [options])
227 
228 Creates new widget and returns it. Options is an object containing the fields x,y,width,height,etc... described below.

230 ### addWidget(el, [x, y, width, height, autoPosition, minWidth, maxWidth, minHeight, maxHeight, id])
231 
232 Creates new widget and returns it.
233 
234 Parameters:
235 
236 - `el` - widget to add
237 - `x`, `y`, `width`, `height` - widget position/dimensions (optional)
238 - `autoPosition` - if `true` then `x`, `y` parameters will be ignored and widget will be places on the first available
239 position (optional)
240 - `minWidth` minimum width allowed during resize/creation (optional)
241 - `maxWidth` maximum width allowed during resize/creation (optional)
242 - `minHeight` minimum height allowed during resize/creation (optional)
243 - `maxHeight` maximum height allowed during resize/creation (optional)
244 - `id` value for `data-gs-id` (optional)
245 
246 Widget will be always placed even if result height is more than actual grid height. You need to use `willItFit` method
247 before calling `addWidget` for additional check.
248 
249 ```javascript
250 $('.grid-stack').gridstack();
251 var grid = $('.grid-stack').data('gridstack');
252 grid.addWidget(el, 0, 0, 3, 2, true);
253 ```

310 ### enableMove(doEnable, includeNewWidgets)
311 
312 Enables/disables widget moving. `includeNewWidgets` will force new widgets to be draggable as per `doEnable`'s value by changing the `disableDrag` grid option. This is a shortcut for:

319

322 ```javascript

456 Modify number of columns in the grid. Will update existing widgets to conform to new number of columns,

491 ### willItFit(x, y, width, height, autoPosition)
492 
493 Returns `true` if the `height` of the grid will be less the vertical constraint. Always returns `true` if grid doesn't
494 have `height` constraint.
495 
496 ```javascript

498  grid.addWidget(newNode.el, newNode.x, newNode.y, newNode.width, newNode.height, true);
499 }
500 else {
501  alert('Not enough free space to place the widget');
502 }
